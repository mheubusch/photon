Input Fields

Input fields allow users to type in text information.

Usage

Use input fields when requiring users to enter a text information like email address, account name, searching keyword… that couldn’t be provided for users to select.

Types

Text Field

A text field can simply store the entered data, or even validate the data and provide hints of how to meet the certain criteria.

Search Field

A search field can either live-search results or present search results after users send a query.

Styles

Light Background

Dark Background

Copy Rules

Error Message

  • Don't say please. Just tell people what to do. "Choose a number that is less than . . . “
  • Be as specific as possible in the message. "Choose a date" or "Choose a time" or "Enter an address” are all better than “Enter a value.”
  • Use words like “choose” or “select” instead of “type” or “key”. We'd rather engage users' minds than their hands, and this also speaks more universally to users accessing the Web with touchscreens, speech-recognition software, or other user input devices.
  • If there are programmatic rules governing valid choices, expose these in the messages where possible. Don't just tell someone that “X” is the nearest valid value, tell them that they must choose values in “X” increments.

Search Field Hint

  • Tell users what objects are going to be searched if possible. “Search + object” is more clear than just saying “Search”.
